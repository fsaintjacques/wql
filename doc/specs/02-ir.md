## Goal

Define all IR types and implement the bytecode encode/decode round-trip in the `wql-ir` crate. This crate is the shared contract between the compiler (Block 5) and the runtime (Block 3). Every type decision made here is load-bearing: changing the binary format after Block 3 ships requires a version bump.

---

## Deliverables

- All types in `crates/wql-ir/src/` compile under `#![no_std]` + `alloc`.
- `encode` (behind `alloc` feature) converts `&[Instruction]` to `Vec<u8>` with a valid program header.
- `Program::from_bytes` parses and validates a `&[u8]` slice into a `Program<'_>`.
- `InstructionIter` walks the bytecode yielding `Instruction` values without allocating (for runtime use).
- `decode` (behind `alloc` feature) round-trips: for every vector of `Instruction` values, `decode(encode(instructions)) == instructions`.
- All 22 instructions are covered by unit tests with hand-crafted bytecode.
- `cargo test -p wql-ir` passes.
- `cargo clippy -p wql-ir` and `cargo clippy -p wql-ir --target wasm32-unknown-unknown` pass with zero warnings.

```rust
#![allow(clippy::module_name_repetitions)]

/// Protobuf wire type identifiers.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
#[repr(u8)]
pub enum WireType {
    Varint = 0,
    I64    = 1,
    Len    = 2,
    I32    = 5,
}

/// Decoding mode for the `DECODE` instruction.
/// Determines how the raw wire bytes are interpreted into a `RegisterValue`.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
#[repr(u8)]
pub enum Encoding {
    /// Unsigned varint; stored as `i64` (reinterpret bit pattern).
    Varint = 0,
    /// Zigzag-encoded varint; stored as `i64`.
    Sint   = 1,
    /// 4-byte little-endian fixed; stored as `i64`.
    I32    = 2,
    /// 8-byte little-endian fixed; stored as `i64`.
    I64    = 3,
    /// Length-prefixed bytes; stored as `RegisterValue::Bytes`.
    Len    = 4,
}

/// Default action applied to any field not matched by a `DISPATCH` arm.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum DefaultAction {
    /// Consume the field bytes; emit nothing.
    Skip,
    /// Emit tag + raw value bytes verbatim to the output buffer.
    Copy,
    /// If `wire_type == LEN`: push new scan window, re-run the program at
    /// `target` offset, emit tag + reframed sub-output.
    /// If `wire_type != LEN`: skip.
    ///
    /// `target` is an absolute byte offset from the start of the bytecode
    /// (after the program header) pointing to a `LABEL` instruction.
    Recurse(u32),
}

/// Field match within a single `DISPATCH` arm.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum ArmMatch {
    /// Match any field with the given field number regardless of wire type.
    Field(u32),
    /// Match only when both the field number and wire type equal the given values.
    FieldAndWireType(u32, WireType),
}

/// A single action inside a `DISPATCH` arm's action sequence.
///
/// Actions are executed left-to-right for a matched field. At most one
/// `Frame` action may appear per arm; if present it must be last.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum ArmAction {
    /// Emit current tag + raw value bytes to the output buffer.
    Copy,
    /// Consume the field bytes; emit nothing.
    Skip,
    /// Decode the current field into register `reg` using `encoding`.
    Decode { reg: u8, encoding: Encoding },
    /// Enter sub-message scope. `target` is an absolute byte offset to a
    /// `LABEL` instruction. The field must have wire type `LEN`.
    Frame(u32),
}

/// A single arm within a `DISPATCH` instruction.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct DispatchArm {
    pub match_:  ArmMatch,
    /// One or more actions executed when this arm matches. The vec is always
    /// non-empty; the encoder must reject empty action lists.
    pub actions: alloc::vec::Vec<ArmAction>,
}

/// The complete WVM instruction set.
///
/// The 22 distinct operations are: `DISPATCH`, `FRAME` (via `ArmAction`),
/// `LABEL`, `COPY`, `SKIP`, `DECODE`, six integer comparisons, `CMP_LEN_EQ`,
/// four `BYTES_*` operations, `BYTES_MATCHES` (optional), `IN_SET`, `IS_SET`,
/// `AND`, `OR`, `NOT`, `RETURN`.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum Instruction {
    // ------------------------------------------------------------------ Control
    /// The only looping instruction. Iterates over all `(field_num, wire_type)`
    /// pairs in the current scan window and routes each to the first matching arm.
    /// Fields matching no arm receive the `default` action.
    Dispatch {
        default: DefaultAction,
        arms:    alloc::vec::Vec<DispatchArm>,
    },

    /// Positional program entry point. No-op at runtime; used by the linker to
    /// resolve `Frame` and `Recurse` targets to absolute byte offsets.
    Label,

    // ---------------------------------------------------------------- Leaf actions
    /// Emit current tag + raw value bytes to the output buffer.
    Copy,

    /// Consume the current field bytes; emit nothing.
    Skip,

    /// Decode the current field into register `reg`.
    Decode { reg: u8, encoding: Encoding },

    // ------------------------------------------- Predicate: integer comparisons
    CmpEq  { reg: u8, imm: i64 },
    CmpNeq { reg: u8, imm: i64 },
    CmpLt  { reg: u8, imm: i64 },
    CmpLte { reg: u8, imm: i64 },
    CmpGt  { reg: u8, imm: i64 },
    CmpGte { reg: u8, imm: i64 },

    // ----------------------------------------------- Predicate: bytes comparisons
    CmpLenEq     { reg: u8, bytes: alloc::vec::Vec<u8> },
    BytesStarts  { reg: u8, bytes: alloc::vec::Vec<u8> },
    BytesEnds    { reg: u8, bytes: alloc::vec::Vec<u8> },
    BytesContains{ reg: u8, bytes: alloc::vec::Vec<u8> },

    /// RE2 regex match. Gated behind the `regex` feature; programs using this
    /// instruction set `FLAG_REGEX_REQUIRED` in the program header.
    #[cfg(feature = "regex")]
    BytesMatches { reg: u8, pattern: alloc::vec::Vec<u8> },

    // ------------------------------------------- Predicate: set / existence
    InSet { reg: u8, values: alloc::vec::Vec<i64> },
    IsSet { reg: u8 },

    // ----------------------------------------------------------------------- Logic
    And,
    Or,
    Not,

    // ----------------------------------------------------------------------- Return
    Return,
}

// ─────────────────────────────────────────── Program

/// Magic bytes at the start of every WVM program: b"WQL\x00".
pub const MAGIC: [u8; 4] = [0x57, 0x51, 0x4C, 0x00];

/// Current bytecode format version.
pub const VERSION: u16 = 1;

/// Header flag: program contains at least one `BYTES_MATCHES` instruction.
/// Runtime must have the `regex` feature enabled to execute such programs.
pub const FLAG_REGEX_REQUIRED: u16 = 0x0001;

/// Fixed-size program header (14 bytes).
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct ProgramHeader {
    /// Must equal `VERSION` for this crate version.
    pub version:         u16,
    /// Number of registers required (0–16).
    pub register_count:  u8,
    /// Maximum `FRAME` nesting depth (informs runtime stack allocation).
    pub max_frame_depth: u8,
    /// Bitfield of `FLAG_*` constants.
    pub flags:           u16,
    /// Length of the bytecode that follows the header, in bytes.
    pub bytecode_len:    u32,
}

/// A parsed, validated WVM program that borrows its backing byte slice.
///
/// The lifetime `'a` is tied to the buffer that contains both the header
/// and the bytecode. Create via `Program::from_bytes`.
pub struct Program<'a> {
    pub header:   ProgramHeader,
    /// Raw bytecode bytes (does not include the 14-byte header).
    pub bytecode: &'a [u8],
}
```

## Binary Format

### Header Layout (14 bytes, fixed)

| Offset | Size | Field | Value |
|--------|------|-------|-------|
| 0 | 4 | magic | `57 51 4C 00` |
| 4 | 2 | version | u16 LE |
| 6 | 1 | register_count | u8 |
| 7 | 1 | max_frame_depth | u8 |
| 8 | 2 | flags | u16 LE |
| 10 | 4 | bytecode_len | u32 LE |

### Primitive Encodings

| Name | Encoding |
|------|----------|
| **uvarint** | Unsigned LEB128 (protobuf-compatible). 1–5 bytes for values ≤ 2³²−1. |
| **svarint** | Zigzag LEB128: `n → (n << 1) ^ (n >> 63)`. Used for `i64` immediates. |
| **bytes** | uvarint length `N` followed by `N` raw bytes. |

### Opcode Table

| Opcode | Mnemonic | Operands |
|--------|----------|----------|
| `0x00` | `DISPATCH` | See §DISPATCH encoding |
| `0x01` | `LABEL` | _(none)_ |
| `0x02` | `COPY` | _(none)_ |
| `0x03` | `SKIP` | _(none)_ |
| `0x04` | `DECODE` | `reg: u8`, `encoding: u8` |
| `0x05` | `CMP_EQ` | `reg: u8`, `imm: svarint` |
| `0x06` | `CMP_NEQ` | `reg: u8`, `imm: svarint` |
| `0x07` | `CMP_LT` | `reg: u8`, `imm: svarint` |
| `0x08` | `CMP_LTE` | `reg: u8`, `imm: svarint` |
| `0x09` | `CMP_GT` | `reg: u8`, `imm: svarint` |
| `0x0A` | `CMP_GTE` | `reg: u8`, `imm: svarint` |
| `0x0B` | `CMP_LEN_EQ` | `reg: u8`, `bytes: bytes` |
| `0x0C` | `BYTES_STARTS` | `reg: u8`, `bytes: bytes` |
| `0x0D` | `BYTES_ENDS` | `reg: u8`, `bytes: bytes` |
| `0x0E` | `BYTES_CONTAINS` | `reg: u8`, `bytes: bytes` |
| `0x0F` | `BYTES_MATCHES` | `reg: u8`, `pattern: bytes` |
| `0x10` | `IN_SET` | `reg: u8`, `count: uvarint`, then `count × svarint` |
| `0x11` | `IS_SET` | `reg: u8` |
| `0x12` | `AND` | _(none)_ |
| `0x13` | `OR` | _(none)_ |
| `0x14` | `NOT` | _(none)_ |
| `0x15` | `RETURN` | _(none)_ |

### DISPATCH Encoding

```
0x00                           -- opcode
default_kind: u8               -- 0=SKIP, 1=COPY, 2=RECURSE
  [if RECURSE] target: uvarint -- absolute bytecode offset of target LABEL
arm_count: uvarint
for each arm:
  match_kind: u8               -- 0=Field, 1=FieldAndWireType
  field_num: uvarint
  [if FieldAndWireType] wire_type: u8
  action_count: uvarint        -- must be ≥ 1
  for each action:
    action_kind: u8            -- 0=COPY, 1=SKIP, 2=DECODE, 3=FRAME
    [if DECODE] reg: u8, encoding: u8
    [if FRAME]  target: uvarint  -- absolute bytecode offset of target LABEL
```

`WireType` encoding (for `FieldAndWireType`): `VARINT=0`, `I64=1`, `LEN=2`, `I32=5` (matches protobuf wire type IDs).

`Encoding` encoding: `VARINT=0`, `SINT=1`, `I32=2`, `I64=3`, `LEN=4`.

### Label Offsets

All `target` fields (in `RECURSE`, `FRAME` arm actions) are **absolute byte offsets** measured from the first byte of the bytecode (i.e., from offset 14 in the full buffer). The target offset points to the `0x01` (`LABEL`) opcode byte of the intended entry point.

The encoder resolves symbolic label names to absolute offsets in a single pass:
1. First pass: record the byte offset of each `Instruction::Label` in the output stream.
2. Second pass: write instructions, substituting recorded offsets for all `Recurse` and `Frame` targets.

## Constraints & Notes

- **No `std` anywhere.** All types and codec logic must compile under `#![no_std]`. `Vec`, `String`, etc. come from `alloc`.
- **`alloc` feature gate.** `encode` and `decode` (which allocate) are behind `#[cfg(feature = "alloc")]`. `InstructionIter`, `Program::from_bytes`, and all type definitions are always available.
- **Register range.** Register indices must be in `0..16`. The encoder rejects values ≥ 16 with a panic (debug) or silent clamp (release is not the correct call here — use `debug_assert!` + documented precondition). The decoder returns `RegisterOutOfRange`.
- **Label resolution.** The encoder performs a two-pass label resolution. Unresolved labels (a `FRAME` or `RECURSE` whose target label does not exist in `instructions`) are a panic — this is a compiler bug, not a user error.
- **`BYTES_MATCHES` opcode** (`0x0F`) is always reserved in the binary format even when the `regex` feature is off. A decoder without the feature active returns `DecodeError::RegexNotSupported` if it encounters opcode `0x0F`.
- **Serde.** The `serde` feature on `wql-ir` is reserved for future tooling (bytecode inspection, snapshot tests). It is not implemented in this block.
- **`IN_SET` with zero elements.** Permitted in the binary format; runtime evaluates it as always-false. The compiler should not emit it, but the codec must not reject it.
- **prost-types dependency** is on `wql-compiler`, not `wql-ir`. The IR crate has no knowledge of protobuf schema types.
